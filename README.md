# УрФУ — абитуриенты по направлениям

Одностраничное приложение для анализа заявлений приёмной кампании УрФУ: конкурс по направлениям, рейтинг по баллам, первый приоритет и позиция «где я» по своему баллу.

**Сайт:** https://omaul.github.io/urfu-entrant/

## Как пользоваться

1. Просто откройте сайт — последний срез загрузится сам. Отметка свежести данных показывается в шапке; срез обновляется автоматически раз в день (~12:00 по Екатеринбургу).
2. Кнопка **«Обновить с сервера»** внизу подтягивает данные с `urfu.ru/api/entrant/` прямо сейчас — текущий вид и фильтры сохраняются. Может не сработать из-за CORS: тогда останется автоматический срез.
3. Всё считается локально в браузере: фильтры по институту, финансированию, статусу, приоритету и баллу, экспорт в CSV.

## Структура

- `index.html` — разметка;
- `css/style.css` — стили;
- `js/config.js` — настройки среза (дата, источник);
- `js/app.js` — логика приложения;
- `assets/urfu_packed.json` — срез данных (фолбэк на случай недоступности urfu.ru при деплое);
- `scripts/update-snapshot.mjs` — скачивает и пакует свежие данные, обновляет дату в `js/config.js`.

Данные никуда не отправляются — приложение полностью статическое.

## Публикация и обновление данных

Workflow `.github/workflows/deploy.yml` деплоит сайт на GitHub Pages при каждом пуше в `main` и по расписанию раз в день. Перед сборкой артефакта он запускает `scripts/update-snapshot.mjs`, который скачивает свежие данные и подкладывает их в публикуемый срез — сами данные в git не коммитятся. Если urfu.ru недоступен, публикуется закоммиченный `assets/urfu_packed.json`.
